Classification Task: Each message's text column was evaluated against the question: "Does this message describe a problem with the delivery of a shipped album or print — such as delivery to the wrong address, lost in transit, stolen package, or returned to sender?"

Result: 608 of 40,967 messages (1.48%) were classified as Yes (delivery issue detected). These span 327 unique Front conversations (based on distinct conversation_id values — see Limitations for caveats on what this means for unique incidents).

Methodology

Classification was performed using a multi-pass keyword and context pattern matching system against the text column:

PassActionResult
1. Email ExclusionRemoved businessprofile-noreply@google.com and mailer-daemon@googlemail.com from recipient_handle451 rows removed → 40,967 remaining
2. Quote StrippingRemoved quoted reply chains (lines starting with >, "On… wrote:" headers, forwarded email blocks) to avoid matching on Nuovo's own outbound language echoed in client repliesPrevents false positives from quoted text
3. Delivery Pattern Matching30+ regex patterns targeting: non-receipt of physical products (album, print, canvas, frame, stainless, wall art), wrong/incorrect address, lost in transit, stolen/theft, returned to sender/unclaimed, damaged in shipping, tracking discrepancies, redelivery requestsInitial matches identified
4. False Positive ExclusionExcluded matches in payment/financial context ("not received payment"), digital access issues ("can't open link"), Nuovo outbound follow-ups ("we haven't received your response/booking"), and outbound messages with only generic (non-product-specific) patternsRemoved 2,376 false positives

Note: This is keyword-based classification, not LLM-based. Accuracy is high for clear-cut cases but may miss nuanced/indirect delivery complaints or occasionally include edge cases. Manual spot-checking of the output is recommended.

Limitations & Recommendations

  • Keyword-based, not LLM-based: Classification uses regex pattern matching on the text column. Indirect or nuanced delivery complaints (e.g., a client describing a problem without using typical keywords) may be missed.
  • Category labels are derived, not raw data: The Issue Category Breakdown (Unclaimed, Non-Receipt, etc.) is produced by which regex patterns matched each message. These categories are not a column in the source Excel. A message may match multiple categories, so category counts sum to more than 608. Re-running the classifier will reproduce these counts, but they are not independently verifiable without manual review.
  • Quoted text stripping: Reply chains are stripped to avoid false positives, but aggressive stripping may occasionally remove relevant client text that appears after a quote block.
  • Both inbound and outbound included: The output includes Nuovo's outbound messages about delivery issues (e.g., redelivery coordination with GTA Imaging). Filter to is_inbound = True if only client-reported issues are needed.
  • Spot-check recommended: Recommend manually reviewing 20–30 random "Yes" and "No" rows to validate accuracy before using in production reporting.
  • Multi-message conversations: A single delivery issue may span multiple messages in the same conversation. The 608 count is messages, not unique incidents. 327 is the exact count of distinct conversation_id values in the 608 flagged messages. This is a better proxy for unique delivery issues than the raw message count, but 327 conversations does not necessarily equal 327 unique incidents — a single client may open multiple conversations about the same issue (inflating the count), and some flagged conversations may be false positives. Validate against GTA Imaging fulfillment data for a definitive incident count.
